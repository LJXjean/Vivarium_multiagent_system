## Ebiten精灵和Server的Agent连接

```
要将 Ebiten 精灵信息保存在服务器端的代理中，并通过指针在 Ebiten 端管理这些精灵，需要确保服务器端和 Ebiten 端共享同一套数据结构。

1. 定义共享的数据结构：在服务器端定义一个代理结构，其中包括需要在 Ebiten 端显示的所有信息，如位置、状态等。
2. 同步和通信机制：使用 WebSocket ，在服务器端和 Ebiten 客户端之间传递数据。
3. 在 Ebiten 端管理精灵：在 Ebiten 端，根据从服务器端接收的数据来创建和更新精灵。每当服务器端的代理状态发生变化时，都应该向 Ebiten 端发送更新。
4. 添加布尔属性检测生物状态：在服务器端的生物代理结构中添加一个布尔属性 IsAlive。当生物不再属于 allOrganismes 集合时，将此属性设置为 false。
5. Ebiten 端移除死亡的精灵：Ebiten 客户端接收到更新时，应检查生物的 IsAlive 属性。如果生物已死亡（IsAlive == false），则从渲染循环中移除相应的精灵。
```
